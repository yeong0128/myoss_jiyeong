# Linux 프로세스 관련 명령어 정리 (top, ps, jobs, kill)

## 1. top
### 1) 하는 일
`top`은 현재 시스템에서 실행 중인 프로세스들을 **실시간으로 모니터링**하는 명령어다.  
CPU 사용률, 메모리 사용량, 프로세스 상태 등을 계속 갱신하면서 보여준다.

### 2) 기본 사용 예시 및 주요옵션
```bash
top -n 5 : 5번 갱신 후 종료
top -u 사용자명 : 특정 사용자의 프로세스만 보기
top -p PID : 지정한 PID만 모니터링
```

### 3) 실행중 단축키
실행중 p  #CPU기준 정렬
실행중 M  #메모리 기준 정렬

## 2. ps
### 1) 하는 일
'ps'는 현재 실행중인 프로세스를 정적으로 한번 출력하는 명령어다.
실시간 갱신이 필요하지 않을때 특정 프로세스 상태를 확인할때사용된다.


### 2) 기본사용예시 및 옵션
```bash
ps aux
ps -ef : 전체 프로세스를 full-format으로 출력
ps aux | grep<이름>
```
| 옵션 | 설명 |
|------|------|
| a | 모든 사용자 프로세스 표시 |
| u | 사용자 기반 상세 정보 |
| x | 터미널 없는 프로세스도 출력 |

## 3. jobs
### 1) 하는일
'jobs'는 현재 셀엣 실행중이거나 백그라운드로 전환된 작업 목록을 보여주는 명령어이다.
ctrl+z로 일시중단하거나 &로 백그라운드 실행한 작업을 관리할 수 있다.

### 2) 기본 사용예시 및 관련 명령어
```bash
jobs
jobs -l
```
fg %번호  #작업을 포그라운드로 실행
bg %번호  #정지된 작업을 백그라운드로 재실행

## 4. kill
### 1) 하는일
'kill'은 특정 프로세스(PID)에 신호를 보내 종료하거나 제어하는 명령어이다.
프로그램이 멈췄을때 강제종료하거나, 정상 종료 요청을 보낼 때 사용된다.

### 2) 기본사용예시 및 옵션
```bash
kill <PID> : 기본종료
kill -9 <PID> : 강제종료
```
| Signal | 설명 |
|--------|------|
| -15 (SIGTERM) | 정상 종료 요청 |
| -9 (SIGKILL) | 즉시 강제 종료 |
| -STOP | 프로세스 일시 중지 |
| -CONT | 중지된 프로세스를 다시 실행 |
